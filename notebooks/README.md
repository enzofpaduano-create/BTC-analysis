# notebooks/

Exploration uniquement. **Aucun code de prod ne doit vivre ici.**

Workflow :
1. Brouillon dans un notebook → on valide visuellement.
2. Si l'idée est gardée, on la porte dans `features/`, `signals/`, etc., **avec tests**.
3. Le notebook reste en référence mais n'est pas importé par le reste du code.

Les notebooks sont gitignorés par défaut quand ils commencent par `scratch_*`.
